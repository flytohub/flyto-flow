# Security Policy

## Supported Version

Security fixes are released for the latest `main` revision and the latest
published release.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability.

Report it through GitHub private vulnerability reporting or email
security@flyto2.com. Include the affected revision, reproduction steps, impact,
and a proof of concept when available.

We target an initial response within 48 hours, severity assessment within seven
days, and coordinated remediation within 90 days for most reports. Critical
issues may require an accelerated private release.

Good-faith security research that respects privacy, avoids service disruption,
and provides reasonable time for remediation will not be pursued legally by
the project.
